【質問タイトル】
JV-Link APIで過去（1年以上前）の「時系列オッズ（0B41）」を取得する方法について

【質問内容】
お世話になります。
Python + JV-Link (win32com) で競馬データ収集アプリを自作しています。
「確定オッズ（RACE詳細のO1レコード）」の取得はできていますが、
**「過去（例: 2020年や2004年）の時系列オッズ（分刻みの推移）」** の取得方法で行き詰まっています。

TARGET frontier JVなどのソフトでは過去の時系列オッズも一括ダウンロードできるため、
API（JV-Link）としても機能自体は提供されているはずと考えていますが、
以下の試行がすべてエラーまたはデータなしとなります。

【試したこと】
1. **蓄積系 JVOpen での取得**
   - コード: `JVOpen(dataspec="0B41", from_time="20200105000000", option=4)`
   - 結果: **エラーコード -111** (パラメータ不正？)
   - 補足: `option=1, 2, 3` でも同様に -111 になります。
   - `0B12`, `0B15`, `0B30` でも同様。

2. **速報系 JVRTOpen での過去再生**
   - コード: `JVRTOpen(dataspec="0B41", key="20200105")`
   - 結果: **戻り値 -1** (データなし)
   - 補足: 直近（先週など）の日付だとデータが取れることは確認済みですが、数ヶ月以上前になると -1 になります。
   - `0B51` を指定すると `0` (成功) が返りますが、取得できるデータがヘッダのみ（1レコード）で実体がありません。

【知りたいこと】
「過去の時系列オッズ」を取得するための、正しい **JVOpenのパラメータ（dataspec名、option番号）** は何になるのでしょうか？
それとも、JVRTOpenに特別なオプションが必要なのでしょうか？

（例: 0B41ではなく別のデータスペックIDがある、あるいは option=4 の使い方が違う、など）

もしご存知の方がいらっしゃいましたら、ヒントだけでも頂けると幸いです。
環境: Windows 11, Python 3.12 (32bit), JRA-VAN Data Lab. SDK Ver4.9.0
