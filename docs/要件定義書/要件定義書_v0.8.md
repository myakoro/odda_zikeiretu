本文書は、JRA-VAN Data Lab SDK (JV-Link) を使用した時系列オッズ取得アプリケーションの要件を定義するものである。本システムは、過去の時系列オッズデータおよびリアルタイムオッズを取得・蓄積・分析し、投資競馬のための意思決定を支援することを目的とする。

## 2. システム概要

### 2.1 システムの目的

本システムは以下の目的を達成するために開発された:

1. **過去データの効率的な取得**: JV-Link APIを使用して、指定期間のレース情報と時系列オッズデータを自動取得
2. **1年以上前のデータ補完**: JV-Link APIの制約(1年以内のデータのみ取得可能)を補うため、ターゲットソフトウェアからエクスポートしたCSVデータをインポート
3. **リアルタイム監視**: 開催日当日のオッズ変動をリアルタイムで監視・記録
4. **データの永続化**: SQLiteデータベースにレース情報とオッズ履歴を保存
5. **データ分析支援**: Webダッシュボードによるデータ可視化とCSVエクスポート機能

## 3. 機能要件

### 3.1 過去データ取得機能 (collector.py)

#### 3.1.1 概要

JV-Link APIを使用して、指定期間内の全レースの時系列オッズデータを取得する。

#### 3.1.2 詳細機能

##### F-001: 期間指定データ取得

| 項目 | 内容 |
|:---|:---|
| 機能ID | F-001 |
| 機能名 | 期間指定データ取得 |
| 説明 | ユーザーが指定した開始日～終了日の範囲内で、全レースの時系列オッズを取得する |
| 入力 | 開始日(YYYYMMDD), 終了日(YYYYMMDD), データ種別("RACE"または"0B41") |
| 出力 | データベースへの保存、処理ログのコンソール出力 |
| 処理フロー | 1. レース開催情報をスキャン<br>2. 有効なレースキーリストを作成<br>3. 各レースの詳細データを取得<br>4. データベースに保存 |

##### F-002: 2段階取得プロセス

1. **第1段階: レーススキャン**
   - データ種別 "RACE" でJVOpenを実行
   - 期間内の全レース情報を取得
   - レースキー(race_id)のリストを作成

2. **第2段階: 詳細データ取得**
   - データ種別 "0B41" (時系列オッズ) でJVRTOpenを実行
   - 各レースキーに対して個別に時系列データを取得
   - RAレコード(レース詳細)とO1レコード(単勝・複勝オッズ)を解析

##### F-003: 取得対象データ

| データ種別 | JV-Dataレコード | 内容 |
|:---|:---|:---|
| レース基本情報 | RAレコード | レースID、開催日、競馬場コード、レース番号、レース名、発走時刻 |
| 単勝オッズ時系列 | O1レコード | 発表時刻、馬番、単勝オッズ、人気順 |
| 複勝オッズ時系列 | O2レコード | 発表時刻、馬番、複勝オッズ下限・上限 (※提供されている場合) |

#### 3.1.3 安定化・リカバリ機能

##### F-004: リトライ処理

- 通信エラーや一時的な失敗が発生した場合、最大3回までリトライを実行
- リトライ間隔: 2.0秒
- 3回失敗した場合は当該レースをスキップし、次のレースへ進む

##### F-005: ハードリセット機能

リトライ時には以下の手順でJV-Link接続を完全にリセット:

1. `JVClose()` でセッションを閉じる
2. JVLinkオブジェクトを `None` に設定
3. `gc.collect()` でガベージコレクションを強制実行
4. 新規にJVLinkオブジェクトを作成
5. `JVInit()` で再接続

**目的**: ドライバ内部で発生した「固まり」や「ゾンビ状態」を解消

##### F-006: 途中再開(レジューム)機能

| 項目 | 内容 |
|:---|:---|
| 記録ファイル | `data/completed_races.log` |
| 記録形式 | `race_id,YYYY-MM-DD` (1行1レース) |
| 再開ロジック | 起動時にログを読み込み、30日以内に取得済みのレースはスキップ |
| 再取得ロジック | 30日以上前に取得したレースは再取得対象とする |

**メリット**:
- 数万レースの処理中にPCがクラッシュしても、直近の箇所から即座に再開可能
- 発走前に取得したレースも、後日自動的に確定オッズまで取得できる

##### F-007: クラッシュ検知・自動回避機能

| 項目 | 内容 |
|:---|:---|
| 一時ファイル | `data/last_attempt.txt` |
| 記録タイミング | レース処理開始時 |
| クラッシュ検知 | 起動時に `last_attempt.txt` が残っている場合、前回そのレースで落ちたと判断 |
| 自動回避 | 該当レースIDを `completed_races.log` に強制追加してスキップ |

**目的**: 特定のレースデータが原因でアプリが無限クラッシュループに陥るのを防ぐ

##### F-008: 欠損データの高速スキップ

以下のJV-Linkエラーコードが返された場合、リトライを行わずに即座にスキップ:

| エラーコード | 意味 | 対応 |
|:---|:---|:---|
| -1 | 汎用エラー(サーバーファイル欠損など) | 即座にスキップ |
| -111 | 取得対象外 | 即座にスキップ |
| -203 | 該当データなし | 即座にスキップ |

**目的**: 中止レースやデータ不備のある期間での無駄な待機時間を削減

### 3.2 ターゲットCSVインポート機能 (import_from_target.py)

#### 3.2.1 概要

JV-Link APIでは1年以上前のデータが取得できないため、ターゲットソフトウェアからエクスポートしたCSVファイルをインポートする機能を提供する。

#### 3.2.2 詳細機能

##### F-009: CSVファイル解析

| 項目 | 内容 |
|:---|:---|
| 機能ID | F-009 |
| 機能名 | ターゲットCSVインポート |
| 対応フォーマット | ターゲット標準エクスポート形式(ワイド形式、Shift-JIS) |
| 入力 | CSVファイルパス (単一ファイルまたはワイルドカード指定) |
| 出力 | データベースへの保存、インポート結果サマリー |

##### F-010: データ変換処理

1. **レースID解析**: `2025120607050101` → `(2025-12-06, 07, 05, 01)`
   - 年月日(8桁) + 競馬場コード(2桁) + 回次(2桁) + 日次(2桁) + レース番号(2桁)

2. **タイムスタンプ変換**: `12051831` → `2025-12-05 18:31`
   - 月日時分(8桁)をレース開催年と組み合わせて完全なタイムスタンプに変換

3. **競馬場コード変換**: 数値コードから競馬場名へのマッピング
   - 例: `01`→札幌, `02`→函館, `03`→福島, `04`→新潟, `05`→東京, `06`→中山, `07`→中京, `08`→京都, `09`→阪神, `10`→小倉

4. **ワイド形式からロング形式への変換**:
   - CSVの横方向に並んだ馬番ごとのオッズを、1行1馬番の形式に変換
   - 各タイムスタンプごとに全馬番のデータを展開

##### F-011: 一括インポート

```powershell
# 単一ファイル
python src/import_from_target.py target系/JD07255101.CSV

# 複数ファイル一括
python src/import_from_target.py "target系/*.CSV"
```

##### F-012: インポート結果サマリー

- 処理ファイル数
- 総レース数
- 総オッズレコード数
- エラー発生数

### 3.3 リアルタイム監視機能 (realtime_monitor.py)

#### 3.3.1 概要

JV-LinkのJVRTOpen APIを使用して、開催日当日のオッズ変動をリアルタイムで監視・記録する。

#### 3.3.2 詳細機能

##### F-013: リアルタイムデータ監視

| 項目 | 内容 |
|:---|:---|
| 機能ID | F-013 |
| 機能名 | リアルタイムオッズ監視 |
| 監視対象 | 当日開催レースのオッズ変動 |
| データ種別 | 0B12 (速報オッズ), 0B31 (単勝・複勝リアルタイム) |
| 更新間隔 | JV-Linkサーバーからのプッシュ通知(約1秒間隔でポーリング) |

##### F-014: 対応データ種別

| データ種別 | 内容 |
|:---|:---|
| 0B12 | 速報オッズ(単勝・複勝) |
| 0B31 | オッズ(単勝・複勝)リアルタイム |
| 0B32 | オッズ(枠連) |
| 0B33 | オッズ(馬連) |
| 0B34 | オッズ(ワイド) |
| 0B35 | オッズ(馬単) |
| 0B36 | オッズ(3連複) |
| 0B37 | オッズ(3連単) |

##### F-015: 監視ループ処理

1. JVGetsでデータ取得(バッファサイズ: 50,000 Byte)
2. レコードタイプ判定(先頭2文字)
3. オッズレコード(O1-O7)の場合、パーサーで解析
4. データベースにコミット
5. 1秒待機して次のデータを取得

##### F-016: 受信データのログ出力

```
[2025-12-23 15:30:45] 受信: 単勝オッズ
[2025-12-23 15:31:12] 受信: 複勝オッズ
[2025-12-23 15:32:03] 受信: 馬連オッズ
```

### 3.4 データ解析機能 (parser.py)

#### 3.4.1 概要

JV-Dataフォーマットのバイナリデータを解析し、構造化データとしてデータベースに保存する。

#### 3.4.2 詳細機能

##### F-017: RAレコード解析

**解析対象データ**:
- レース識別情報: 開催日、競馬場コード、レース番号
- レース詳細情報: レース名、発走時刻
- エンコーディング: Shift-JIS

**出力**:
- レースID、開催日、競馬場コード、レース番号、レース名、発走時刻をデータベースに保存

##### F-018: O1レコード解析(単勝・複勝オッズ)

**解析対象データ**:
- レース識別情報: レースID
- オッズ発表時刻: 月日時分
- 馬番ごとのオッズ情報: 馬番、単勝オッズ、人気順(最大28頭)

**出力**:
- レースID、発表時刻、馬番、単勝オッズ、人気順をデータベースに保存
- 単勝オッズは10倍値として格納されているため、10で割って実際のオッズ値に変換

##### F-019: 安全な型変換

- `_safe_float()`: 非数値文字列を0.0に変換、記号(`*`, `-`, 空白)を除去
- `_safe_int()`: 非数値文字列を0に変換、記号を除去
- エンコーディング: Shift-JIS、エラー時は `replace` モードでデコード

## 4. 非機能要件

### 4.1 性能要件

| 項目 | 要件 |
|:---|:---|
| データ取得速度 | 1レースあたり平均1秒以内(ネットワーク状況に依存) |
| リアルタイム更新間隔 | 1秒以内 |
| データベース応答時間 | 100ms以内(通常のクエリ) |
| Webダッシュボード初期表示 | 3秒以内 |

### 4.2 安定性要件

| 項目 | 要件 |
|:---|:---|
| リトライ回数 | 最大3回 |
| レース間待機時間 | 0.6秒 (ドライバ安定性確保) |
| バッファサイズ | 過去データ: 40,000 Byte<br>リアルタイム: 50,000 Byte |
| メモリ管理 | 定期的なガベージコレクション実行 |

**待機時間の重要性**:
- 0.1秒: JVGetsがハング、処理停止
- 0.3秒: エラー発生(80レコードに1回程度)
- **0.6秒: エラーなし、安定動作** ← 推奨値

### 4.3 可用性要件

| 項目 | 要件 |
|:---|:---|
| クラッシュリカバリ | 自動再開機能により、手動介入なしで復旧可能 |
| データ整合性 | トランザクション管理により、処理中断時のデータロスを最小化 |
| ログ出力 | 全処理の進捗とエラーをコンソールに出力 |

### 4.4 セキュリティ要件

| 項目 | 要件 |
|:---|:---|
| サービスキー管理 | 環境変数 `.env` ファイルで管理、Gitには含めない |
| データベースアクセス | ローカルファイルベース、外部アクセス不可 |
| Web APIアクセス | localhost のみ、外部公開不可 |

### 4.5 保守性要件

| 項目 | 要件 |
|:---|:---|
| コード可読性 | Docstring、コメントによる説明 |
| モジュール分離 | 機能ごとに独立したPythonファイル |
| エラーハンドリング | try-except による例外処理、エラーログ出力 |

## 5. 制約事項

### 5.1 技術的制約

| 項目 | 制約内容 |
|:---|:---|
| OS | Windows 10/11のみ (JV-Link COM依存) |
| Python | 32bit版必須 (JV-Link ActiveXは32bit専用) |
| JV-Link | JRA-VAN契約とSDKインストールが必須 |
| データ取得期間 | JV-Link APIは1年以内のデータのみ取得可能 |
| メンテナンス時間 | JRA-VANメンテナンス時(月曜深夜など)は取得不可 |

### 5.2 運用制約

| 項目 | 制約内容 |
|:---|:---|
| ネットワーク | インターネット接続必須 |
| ディスク容量 | 時系列データは大容量になるため、十分な空き容量が必要 |
| 同時実行 | collector.pyとrealtime_monitor.pyの同時実行は非推奨 |

## 6. ユースケース

### 6.1 過去データ一括取得

**アクター**: ユーザー

**事前条件**:
- JV-Link SDKがインストール済み
- サービスキーが設定済み

**基本フロー**:
1. `アプリ起動.bat` を実行
2. メニューから「1. 過去データを取得する」を選択
3. 開始日と終了日を入力
4. データ種別を選択(RACE または 0B41)
5. 自動的にデータ取得が開始される
6. 進捗がコンソールに表示される
7. 完了後、データベースに保存される

**代替フロー**:
- 処理中にエラーが発生した場合、自動的にリトライ
- 3回失敗した場合、該当レースをスキップして次へ進む
- PCがクラッシュした場合、再起動後に同じ期間を指定すれば自動的に途中から再開

### 6.2 ターゲットCSVインポート

**アクター**: ユーザー

**事前条件**:
- ターゲットソフトウェアから時系列オッズCSVをエクスポート済み
- CSVファイルを `target系/` フォルダに配置

**基本フロー**:
1. コマンドプロンプトを開く
2. 仮想環境をアクティベート
3. `python src/import_from_target.py "target系/*.CSV"` を実行
4. インポート進捗が表示される
5. 完了後、サマリーが表示される

### 6.3 リアルタイム監視

**アクター**: ユーザー

**事前条件**:
- JV-Link SDKがインストール済み
- サービスキーが設定済み
- 開催日当日

**基本フロー**:
1. `アプリ起動.bat` を実行
2. メニューから「2. リアルタイム監視を開始する」を選択
3. 監視が開始され、オッズ更新がリアルタイムで表示される
4. Ctrl+C で監視を停止

### 6.4 Webダッシュボードでのデータ分析

**アクター**: ユーザー

**事前条件**:
- データベースにデータが保存済み

**基本フロー**:
1. `python src/api_server.py` を実行
2. ブラウザで `http://localhost:5000` にアクセス
3. ダッシュボードで統計情報を確認
4. レース一覧から分析対象レースを選択
5. オッズ分析画面でグラフを確認
6. 必要に応じてCSVエクスポート

## 7. データフロー

### 7.1 過去データ取得フロー

```
[ユーザー入力]
    ↓
[collector.py]
    ↓ JVOpen("RACE")
[JV-Link API] → レース一覧取得
    ↓
[レースキーリスト作成]
    ↓
[各レースに対してループ]
    ↓ JVRTOpen("0B41", race_key)
[JV-Link API] → 時系列オッズ取得
    ↓ JVGets
[RAレコード、O1レコード]
    ↓
[parser.py] → データ解析
    ↓
[SQLite Database] → 保存
    ↓
[completed_races.log] → 完了記録
```

### 7.2 ターゲットCSVインポートフロー

```
[ターゲットCSV]
    ↓
[import_from_target.py]
    ↓
[CSV読み込み (Shift-JIS)]
    ↓
[レースID解析]
    ↓
[タイムスタンプ変換]
    ↓
[ワイド形式→ロング形式変換]
    ↓
[SQLite Database] → 保存
    ↓
[インポートサマリー表示]
```

### 7.3 リアルタイム監視フロー

```
[realtime_monitor.py]
    ↓ JVRTOpen("0B12")
[JV-Link API] → リアルタイム接続
    ↓
[監視ループ]
    ↓ JVGets (1秒間隔)
[オッズ更新データ]
    ↓
[parser.py] → データ解析
    ↓
[SQLite Database] → 保存
    ↓
[コンソール出力]
```

## 8. 用語集

| 用語 | 説明 |
|:---|:---|
| JV-Link | JRA-VAN Data Lab SDKが提供するCOM Object API |
| JVOpen | 過去データ取得用API |

| O1レコード | 単勝・複勝オッズレコード |
| レースキー | レースを一意に識別するID (YYYYMMDDJJRR形式) |
| ターゲット | 競馬予想ソフトウェア (株式会社ホースメンズ) |
| 時系列オッズ | 発走前の各時刻におけるオッズ変動データ |
| 確定オッズ | 発走直前の最終オッズ |
